# 🚢 מיק (the-mic) — מנהל משימות Gmail→Sheets

מנהל משימות שהופך מיילים מ-Gmail לטבלת משימות חיה ב-Google Sheets — עם סנכרון דו-כיווני, צביעה אוטומטית,
מייל סיכום יומי, ו-**Web App אינטראקטיבי** (לוח / רשימה / Roadmap, מיילסטונז, תאריכי התחלה, מצב כהה).

> **TL;DR**
> שולחים לעצמך מייל עם נושא במבנה `קטגוריה - משימה` (למשל `שיווק - להכין מצגת`).
> פילטר ב-Gmail מדביק לו אוטומטית תגית `MyTasks`.
> בלחיצת כפתור בגיליון, כל המיילים האלה הופכים לשורות בטבלה — צבועות, מסודרות, ומסונכרנות לשני הכיוונים עם תיבת המייל.

---

## 📋 מה הסקריפט עושה

- **מושך מיילים** עם התגית `MyTasks` והופך כל אחד לשורה בגיליון בשם **"משימות"**.
- **קורא את נושא המייל** ומפצל אותו לפי המבנה `קטגוריה - משימה`. לדוגמה, הנושא `לקוחות - לחזור לדני` ייכנס כך:
  - קטגוריה = `לקוחות`
  - משימה = `לחזור לדני`
- **מסנן אוטומטית** מה שלא רלוונטי: נושאים בלי טקסט בעברית, הזמנות ליומן, נושאים ארוכים או קצרים מדי, ונושאים בלי המבנה הנכון.
- **סנכרון דו-כיווני** בין הגיליון ל-Gmail:
  - סימון משימה כ-`DONE` בטבלה → המייל המתאים עובר **לאשפה** ב-Gmail.
  - הסרת התגית `MyTasks` מהמייל ב-Gmail → המשימה מסומנת אוטומטית `DONE` בטבלה.
- **צביעה אוטומטית**:
  - כל קטגוריה מקבלת **צבע קבוע משלה** (נשמר בגיליון נפרד בשם "צבעים").
  - עמודת הסטטוס נצבעת לפי המצב: `OPEN` (כחול בהיר), `IN PROGRESS` (כחול כהה), `DEPENDENT` (אפור), `DONE` (ירוק).
- **מונע כפילויות** — מייל שכבר קיים בטבלה לא ייכנס שוב.

## 🗂️ מבנה הטבלה

הגיליון "משימות" נוצר אוטומטית עם העמודות הבאות:

| עמודה | שם | תיאור |
|-------|----|-------|
| A | Task ID | מזהה ייחודי (`קטגוריה - משימה`). **מוסתרת** — לשימוש פנימי; אף פעם לא משתנה (מונע כפילויות). |
| B | קטגוריה | החלק שלפני ה-` - ` בנושא המייל. |
| C | משימה | החלק שאחרי ה-` - `. |
| D | סטטוס | תפריט נפתח: `OPEN` / `IN PROGRESS` / `DEPENDENT` / `DONE`. |
| E | עדיפות | `P0`–`P3` (או טקסט חופשי). |
| F | תאריך יעד | תאריך. |
| G | הערות | חופשי. |
| H | Thread ID | מזהה שרשור המייל ב-Gmail. **מוסתרת** — מחברת בין השורה למייל. |
| I | תאריך התחלה | אוטומטי ביצירה / מתאריך המייל; משמש את תצוגת ה-Roadmap. |
| J | Milestone ID | שיוך למיילסטון (גיליון "Milestones"). **מוסתרת**. |

בנוסף נוצרים אוטומטית גיליונות עזר: **"צבעים"** (צבע לכל קטגוריה) ו-**"Milestones"** (`id, name, start, end, color, project`).

> ⚠️ אל תמחק/תשנה ידנית את העמודות המוסתרות (A, H, J) — הן מה שמאפשר את הסנכרון והשיוכים.
> 💡 עמודות I/J נוספו **אחרי** Thread ID בכוונה — כך הוספתן לא הזיזה נתונים קיימים.

## ⚙️ התקנה (פעם אחת)

1. פתח **Google Sheet** חדש (או קיים).
2. בתפריט העליון: **תוספים (Extensions)** → **Apps Script**.
3. מחק כל קוד קיים, והדבק את התוכן של [`syncGmailToSheets.TXT`](./syncGmailToSheets.TXT).
4. לחץ על **💾 שמור** (אייקון הדיסקט).
5. חזור לגיליון ו**רענן את הדף** (F5). יופיע בתפריט העליון פריט חדש: **"ניהול משימות"**.
6. לחץ **ניהול משימות** → **🔄 סנכרן מיילים (Refresh)**. בהרצה הראשונה Google יבקש **הרשאות** — אשר את הגישה ל-Gmail ול-Sheets (זה רגיל, הקוד רץ רק בחשבון שלך).

## 🧰 מסלול חלופי: הקמה דרך `clasp` (טרמינל) — כשמסלול הדפדפן חסום

**מתי צריך את זה:** לפעמים התוסף **Claude in Chrome** נכנס למצב שבו הוא שולט רק ב-origin **אחד** שכבר טעון, וחוסם ניווט/פעולה על כל דומיין אחר — כולל `docs.google.com`, `script.google.com` ו-`mail.google.com` — עם השגיאה `Navigation to this domain is not allowed`. זה קורה **גם** כשהרשאת Chrome היא "On all sites", **וגם** אחרי אתחול התוסף — כי ההגבלה היא פר-origin ומאוחסנת.

> **בדיקה מהירה (10 שניות):** בקש מ-Claude לנווט ל-`example.com`. אם *גם* זה חסום — זו החסימה הגלובלית פר-origin, לא בעיה ספציפית ל-Google. אל תבזבז זמן על אבחון — עבור ישר ל-clasp.

**היתרון:** `clasp` (הכלי הרשמי של Google ל-Apps Script) מקים את כל צד-הקוד מהטרמינל — בלי להדביק ~95KB קוד לעורך, ובלי תלות בדפדפן. רק ה-OAuth וכמה קליקים ב-Gmail/Sheet דורשים אותך.

```bash
# 1) התקנה + התחברות (בחר את חשבון Google הנכון בדפדפן שנפתח)
npm install -g @google/clasp
clasp login                       # השתמש בזרימת ה-localhost הרגילה (לא --no-localhost)

# 2) הדלק את ה-Apps Script API (פעם אחת):
#    https://script.google.com/home/usersettings  →  הדלק את המתג

# 3) פרויקט Apps Script מקושר לגיליון חדש בשם "מיק"
mkdir mik-clasp && cd mik-clasp
clasp create --type sheets --title "מיק"

# 4) הכנס את הקוד (מהריפו הזה) + ערוך את המניפסט
cp /path/to/the-mic/syncGmailToSheets.TXT ./Code.js
cp /path/to/the-mic/WebApp.html          ./WebApp.html
#   ב-appsscript.json: קבע timeZone (למשל "Asia/Jerusalem") והוסף בלוק "webapp" (ראה למטה)
clasp push -f
```

**פריסת שתי ה-Web Apps — הטריק החשוב:** clasp לוקח את הגדרת הגישה מ-`appsscript.json`, וכל `clasp deploy` "מצלם" גרסה נעולה. לכן כדי לקבל **שתי גישות שונות**, ערוך את `webapp.access` **בין** הפריסות:

```jsonc
// webhook (הסוד MIK_URL): חייב ANYONE_ANONYMOUS — כי סקריפט mik שולח curl בלי אימות
"webapp": { "executeAs": "USER_DEPLOYING", "access": "ANYONE_ANONYMOUS" }
```
```bash
clasp push -f && clasp deploy -d "mik webhook"     # → זו כתובת ה-MIK_URL
#   עכשיו שנה במניפסט ל:  "access": "MYSELF"
clasp push -f && clasp deploy -d "mik UI"          # → זו כתובת ה-MIK_APP_URL
```
כתובת ה-`/exec` לכל פריסה נבנית כך: `https://script.google.com/macros/s/<DEPLOYMENT_ID>/exec`.

**אישור + אימות:** פתח את הגיליון החדש → **ניהול משימות → 🔄 סנכרן מיילים** → אשר OAuth. זה **הכרחי** — ה-webhook רץ בהרשאות המשתמש שפרס, ולא יעבוד עד שתאשר פעם אחת. אימות מהיר של ה-webhook:
```bash
curl -s --data-urlencode "task=בדיקה - התקנה" "<MIK_URL>"    # ציפייה: {"ok":true,...}
```
משם ממשיכים רגיל: תגית + פילטר ב-Gmail (למטה), ו-`desktop/install.sh`.

> 🧭 **גוצ'ות שנתקלנו בהן:**
> - `clasp create` נכשל עם *"User has not enabled the Apps Script API"* → הדלק ב-usersettings וחכה ~דקה להתפשטות.
> - `clasp login --no-localhost` דורש הדבקת קוד ידנית בטרמינל (לא מתאים להרצה ברקע) → השתמש ב-`clasp login` הרגיל; שרת ה-localhost תופס את הקוד לבד.
> - **אל תחיל את פילטר ה-Gmail אחורנית** ("apply to matching conversations") — זה יסחף כל מייל-לעצמך היסטורי לתוך הלוח כמשימות.
> - הגיליון הריק שנוצר בטעות (אם ניסית קודם דרך הדפדפן) — אפשר למחוק ידנית; clasp יוצר גיליון משלו.

## 🌐 שפה: עברית / אנגלית — שאלת התקנה

מיק תומך ב**עברית (ברירת מחדל) ובאנגלית מלאה**. השפה נשמרת ב-Script Property בשם `APP_LANG` (`he` / `en`).

- **בהתקנה:** שאל את המשתמש *"עברית או אנגלית?"*. לאנגלית — קבע `APP_LANG=en` (Apps Script → **Project Settings ⚙️ → Script Properties → Add** → `APP_LANG` = `en`). בלי הגדרה = עברית.
- **מתג בלוח:** כפתור **🌐** בסרגל העליון מחליף שפה בכל רגע (נשמר ל-`APP_LANG`, והדף נטען מחדש בשפה החדשה) — נוח גם לבדיקה.
- **מה מושפע:** כל ה-UI (לוח/רשימה/Roadmap/מודלים/toasts), פריטי התפריט בגיליון, **מייל הסיכום היומי**, וסינון המייל — במצב `en` הסינון מקבל נושאים באותיות לטיניות במקום עברית.

> 🧩 **טכני:** העברית היא מקור-האמת; אנגלית מיושמת כשכבת-תרגום מרוכזת (מילון `he→en` + `translatePage()`), כך שהמצב העברי לא מושפע. הרחבה/תיקון של תרגום = עריכת המילון `DICT` ב-`WebApp.html`.

למיק יש **שלושה מקורות** להזנת משימות, ולא חייבים את כולם. בהתקנה חדשה כדאי שהמשתמש יבחר:

| מקור | דורש הגדרת Gmail? | מתי מתאים |
|---|---|---|
| ✉️ **מייל לעצמך** (תגית `MyTasks`) | ✅ כן (תגית + פילטר) | אוהב לרשום משימות מהמייל |

| ⚡ **בר עליון / `mik` בטרמינל** (webhook) | ❌ לא | לכידה מהירה תוך כדי עבודה |

> **שאלת ההתקנה:** *"איך תרצה להזין משימות — גם דרך מייל-לעצמך, או רק הזנה ישירה (לוח/בר)?"*
> - **רק הזנה ישירה** → אפשר **לדלג על כל סעיף "הגדרת Gmail"** למטה. הגיליון נוצר אוטומטית בהזנה הישירה הראשונה (＋ בלוח / webhook), בלי צורך בסנכרון מייל.
> - **גם מייל** → בצע את הגדרת ה-Gmail למטה.

> ℹ️ בכל מקרה, בהתקנה תתבקש הרשאת Gmail ב-OAuth (הקוד מכיל `GmailApp`) — זה תקין גם אם לא תשתמש במייל. הסנכרון הדו-כיווני והמעבר-לאשפה רלוונטיים רק למסלול המייל.

## ⭐ הגדרת Gmail — תגית אוטומטית **(אופציונלי)**

צריך את זה **רק אם בחרת בהזנה דרך מייל**: שכל מייל "משימה" יקבל אוטומטית את התגית `MyTasks`. עושים את זה פעם אחת.

### שלב 1 — יצירת התגית
1. ב-Gmail, בתפריט הצדדי השמאלי, גלול למטה ולחץ **עוד (More)** → **צור תווית חדשה (Create new label)**.
2. קרא לה בדיוק `MyTasks` (חשוב — הסקריפט מחפש את השם הזה).

### שלב 2 — יצירת הפילטר האוטומטי
1. בתיבת החיפוש של Gmail למעלה, לחץ על אייקון **אפשרויות החיפוש** (המחוונים מימין לתיבה).
2. בשדה **"מכיל את המילים" (Has the words)**, הדבק את הקריטריון הבא:
   ```
   from:me to:me -{cc:me bcc:me}
   ```
   המשמעות: "מיילים שאני שלחתי, אליי עצמי, ושאני לא מופיע בהם גם ב-CC/BCC" — כלומר מיילים שאתה שולח לעצמך בלבד.
3. לחץ **צור מסנן (Create filter)**.
4. בחלון הבא סמן:
   - ✅ **דלג על תיבת הדואר הנכנס / ארכב אותו (Skip the Inbox)** — כדי שהמשימות לא יציפו את ה-Inbox.
   - ✅ **החל את התווית (Apply the label):** בחר `MyTasks`.
5. לחץ **צור מסנן (Create filter)**.

מעכשיו, כל מייל שתשלח לעצמך יקבל אוטומטית את התגית `MyTasks` ויהיה מוכן להיכנס לטבלה בסנכרון הבא.

> 💡 **טיפ:** אפשר גם להוסיף את התגית `MyTasks` ידנית לכל מייל קיים (פתח את המייל → אייקון התגיות → סמן `MyTasks`).

> ℹ️ **מגבלה קטנה שכדאי להכיר:** הפילטר תופס מיילים שאתה שולח לעצמך, אבל הוא **לא** חוסם מצב שבו הוספת אדם *אחר* בשדה ה-To. בשימוש הרגיל ("מייל לעצמי בלבד") זה לא בעיה.

## 🌐 תצוגת Web App (אפליקציית משימות)

מעבר לגיליון, יש **דף web אינטראקטיבי** עם עיצוב מודרני — פותחים אותו מהמחשב או מהטלפון (אפשר להוסיף למסך הבית כמו אפליקציה). הוא קורא וכותב לאותה טבלת "משימות", כך שכל הסנכרון עם Gmail ממשיך לעבוד.

**מה אפשר לעשות בו:**
- **שלוש תצוגות**: **לוח** (קנבן), **רשימה**, ו-**Roadmap** (ציר זמן).
- **תצוגת Roadmap**: שורות לפי קטגוריה או מיילסטון, בר לכל משימה מתאריך התחלה→יעד, אחוז התקדמות לכל שורה, קו "היום", וקבוצת **"לא משויך"** שאפשר לגרור ממנה לתוך מיילסטון.
- **מיילסטונז** (כפתור ◇): יצירה/מחיקה, ושיוך משימות **מהמשימה** (בורר על הכרטיס) / **בבאלק** / **בגרירה** ב-Roadmap.
- **קיבוץ גמיש**: לפי **סטטוס / תאריך (היום·מחר·השבוע·באיחור) / עדיפות / קטגוריה / מיילסטון**.
- **פילטרים מרובי-בחירה**: כמה קטגוריות + כמה סטטוסים בו-זמנית, עם **חיפוש בתוך הקטגוריות**, חיפוש חופשי, מיון, וטוגל "הסתר שהושלמו".
- **עריכה inline על הכרטיסים**: כותרת, קטגוריה, עדיפות (P0–P3), סטטוס, **תאריך התחלה + תאריך יעד**, והערות (תצוגה מקדימה נטויה + לחיצה לעריכה).
- **הוספת משימה ידנית** (＋ משימה) — בלי תלות במייל; מקבלת תאריך התחלה אוטומטי.
- **🗓️ הוספת אירוע ליומן Google** — כפתור **🗓️** על כל כרטיס (ליד ה-📩) פותח מודאל קטן: כותרת (מוזנת מהמשימה), תאריך (מתאריך היעד אם קיים, ניתן לעריכה), טוגל **"יום שלם"**, שעת התחלה + משך, ובורר **לוח שנה**. לחיצה על **"קבע בלוח השנה"** יוצרת את האירוע ביומן שבחרת, ומוסיפה שורת-סימון עם קישור להערות המשימה. *(דורש אישור Calendar חד-פעמי — ראה למטה.)*
- **גרירה ושחרור** בין עמודות הלוח לשינוי סטטוס/עדיפות/קטגוריה/תאריך/מיילסטון.
- **Undo** לסימון "הושלם" (מגן מפני מחיקת המייל בטעות), **פעולות מרובות** (בחירה + שינוי סטטוס/מיילסטון במכה), **קישור 📩 למייל המקורי**.
- **טעינה מהירה**: רק המשימות הפעילות נטענות בהתחלה; משימות שהושלמו (DONE) נטענות על דרישה (כשמכבים "הסתר שהושלמו"). הצ'יפ "הושלמו" מציג ספירה בלי לטעון אותן.
- שורת **סיכום** לחיצה (פתוחות / בתהליך / תלויות / באיחור / הושלמו) שמסננת.
- **מצב כהה**, טוגל צפיפות, קיצורי מקלדת (`/` חיפוש, `C` משימה חדשה, `Esc` סגירה).
- כפתורי **🔄 סנכרן** ו**📧 סיכום** — ישירות מתוך הדף.

### התקנה (פעם אחת)
1. ב-Apps Script, ליד **Files** לחץ **➕** → **HTML**, וקרא לקובץ בדיוק `WebApp` (בלי סיומת — Apps Script מוסיף `.html` לבד).
2. הדבק לתוכו את התוכן של [`WebApp.html`](./WebApp.html) ושמור.
3. ודא שהקוד הראשי ([`syncGmailToSheets.TXT`](./syncGmailToSheets.TXT)) מעודכן (הוא מכיל את `doGet` ופונקציות ה-`web...`).
4. למעלה מימין: **Deploy** → **New deployment** → גלגל השיניים → **Web app**.
5. הגדר:
   - **Execute as:** `Me` (הסקריפט ירוץ בהרשאות שלך).
   - **Who has access:** `Only myself` (מומלץ — רק אתה, הכי בטוח). אפשר גם `Anyone with the link` אם תרצה לשתף.
6. **Deploy** → אשר הרשאות → **העתק את כתובת ה-Web App**.
7. כדי לראות את הכתובת שוב בכל רגע: בגיליון, תפריט **ניהול משימות** → **🌐 הצג כתובת ה-Web App**.

> ⚙️ אחרי שתעדכן את הקוד בעתיד, צריך **Deploy → Manage deployments → ✏️ → Version: New version → Deploy** כדי שהשינויים יופיעו ב-Web App. (במסלול clasp: `clasp push` → `clasp create-version` → `clasp redeploy <deploymentId> -V <version>` — כדי לשמור על **אותה כתובת**.)

> 🗓️ **פיצ'ר "הוסף אירוע ליומן" — אישור חד-פעמי:** הפיצ'ר משתמש ב-`CalendarApp`, שמוסיף הרשאת **Calendar**. בפעם הראשונה: בגיליון → **ניהול משימות → 📅 אשר גישה ליומן (פעם אחת)** → אשר את הבקשה (כולל מסך "unverified app" של הסקריפט שלך עצמך). רק אחרי זה כפתור ה-🗓️ בלוח יעבוד. בורר "לוח שנה" מציג את היומנים שבבעלותך (לכתיבה).

### 🔒 אבטחה — חשוב לקרוא
מומלץ לפרוס עם **"Only myself"** — אז רק אתה (מחובר לחשבון שלך) יכול לפתוח את הכתובת. אם בכל זאת
תבחר **"כל מי שיש לו את הלינק"**, שים לב: **כל מי שמקבל את הכתובת יכול לראות ולערוך את המשימות שלך**
(ולהריץ סנכרון / לשלוח לעצמך סיכום) — אל תפרסם אותה במקום ציבורי.

אם שיתפת בלינק ותרצה שכבת הגנה קלה — אפשר להפעיל **טוקן**:
1. ב-Apps Script: **Project Settings (⚙️)** → גלול ל-**Script Properties** → **Add property**.
2. שם: `ACCESS_TOKEN`, ערך: מחרוזת סודית שתבחר (למשל `my-secret-123`).
3. מעכשיו הדף ייפתח רק עם הכתובת בתוספת `?token=my-secret-123` בסוף. (הסר את ה-property כדי לכבות.)

> 🔐 הטוקן נשמר ב-Script Properties בלבד — **לא** בקוד שבגיט.

## 🖥️ לכידה מהירה מהדסקטופ (CLI + שורת תפריטים)

לרישום משימה תוך כדי היום בלי לפתוח את האפליקציה — `mik "קטגוריה - משימה"` מכל טרמינל, או פריט **⚡** בשורת התפריטים. הקבצים תחת [`desktop/`](./desktop).

**איך זה עובד:** הסקריפט פונה ל-`doPost` (webhook) שבקוד הראשי, על **deployment נפרד מסוג "Anyone"**. **מודל secret-URL** — כתובת ה-`/exec` הארוכה והאקראית היא הסוד (כמו webhook של Slack), אין טוקן נפרד, וה-endpoint **מוסיף-בלבד**. ה-Web App הראשי נשאר "Only myself" ולא מושפע. ברירת מחדל: תאריך התחלה + יעד = היום.

### התקנה במחשב חדש
1. **צור webhook deployment** (פעם אחת): בעורך Apps Script → **Deploy → New deployment → Web app** → **Execute as: Me**, **Who has access: Anyone** → Deploy → העתק את ה-Web app URL (זה הסוד שלך — שמור פרטי).
2. **הרץ את המתקין:**
   ```bash
   git clone https://github.com/Zeev-L/the-mic.git
   cd the-mic && ./desktop/install.sh
   ```
   הוא מעתיק את `mik` + תוסף ה-SwiftBar ל-`~/.mik/`, יוצר `~/.mik/config` (chmod 600), מוסיף alias `mik` ל-`~/.zshrc`, ואם SwiftBar מותקן — מקשר את התוסף.
3. **ערוך את `~/.mik/config`** והדבק את כתובת ה-webhook (`MIK_URL`) ואת כתובת ה-UI הראשי (`MIK_APP_URL`).
4. אם השתמשת ב-SwiftBar — **אתחל אותו** (יצא ופתח מחדש) כדי שיסרוק את התוסף החדש. ⚡ יופיע בשורת התפריטים.

> 💡 **אין SwiftBar?** התקן: `brew install --cask swiftbar`. כדי לחסוך בחירת תיקייה ידנית, הגדר מראש את תיקיית התוספים וקשר את התוסף:
> ```bash
> defaults write com.ameba.SwiftBar PluginDirectory "$HOME/.mik/swiftbar-plugins"
> mkdir -p "$HOME/.mik/swiftbar-plugins"
> ln -sf "$HOME/.mik/swiftbar-mik.sh" "$HOME/.mik/swiftbar-plugins/mik.1h.sh"   # הסיומת .1h.sh חובה
> open -a SwiftBar
> ```
> בפעם הראשונה שתוסיף משימה, macOS יבקש אישור **התראות** ל-SwiftBar — אשר.

### שימוש
- **טרמינל:** `mik "לקוחות - לחזור לדני"` (פתח טרמינל חדש אחרי ההתקנה לטעינת ה-alias).
- **שורת תפריטים:** ⚡ → **＋ הוסף משימה** → תיבה → Enter.
- **קיצור מקלדת (אופציונלי):** הגדר ב-Raycast/Alfred/Shortcuts פעולה שמריצה `mik "..."`.

> 🔒 **אבטחה:** הקובץ `~/.mik/config` הוא הסוד (מכיל את כתובת ה-webhook). הרשאות 600, ולא נכנס לגיט (ה-`.gitignore` חוסם `desktop/config` ו-`.mik/`). ב-repo יש רק `desktop/config.example` עם placeholders.

## 🛠️ פתרון תקלות (FAQ)

**המייל לא נכנס לטבלה אחרי סנכרון.** בדוק שלושה דברים:
1. הנושא בנוי כ-`קטגוריה - משימה` (עם רווח-מקף-רווח באמצע).
2. יש בנושא טקסט **בעברית** (הסקריפט מתעלם ממיילים בלי עברית).
3. למייל יש את התגית `MyTasks`.
> שים לב: הקטגוריה (החלק שלפני ה-` - `) חייבת להיות באורך 2–25 תווים, בלי `:` ובלי המילה `invitation`. זה הסינון נגד הזמנות יומן וזבל.

**הכפתור נותן שגיאה.** שם הפונקציה הראשית חייב להישאר `syncGmailToSheetsV2` — אל תשנה אותו, התפריט מצביע עליו.

**משימה נעלמה / סומנה DONE לבד.** זה תקין — קורה כשהורדת את התגית `MyTasks` מהמייל. הסקריפט מפרש את זה כ"המשימה הסתיימה".

**מהו הגיליון "צבעים"?** זה גיליון עזר שהסקריפט יוצר לבד. הוא שומר איזה צבע שייך לכל קטגוריה (כדי שהצבעים יישארו קבועים). אפשר לערוך שם צבע ידנית, אבל **אל תמחק** את הגיליון.

**מהי הפונקציה `backfillDemoDates`?** כלי **חד-פעמי לדמו** בלבד — ממלא תאריכי התחלה (ויעד היכן שחסר)
פיקטיביים לכל המשימות, כדי שתצוגת ה-Roadmap תיראה מלאה. ממלא **רק תאים ריקים** (לא דורס נתונים),
ובטוח להרצה חוזרת. להרצה: בעורך Apps Script בחר `backfillDemoDates` בתפריט הפונקציות ולחץ Run.
אפשר למחוק אותה אחרי שהדמו מוכן.
